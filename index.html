What does it sound like to a screen reader?

Here are some examples of a label and an input in various states, and what they sound like to a screen reader user.

The exact words are lifted from VoiceOver on Mac OS, but are good for a general guide. A full stop "." denote a slightly longer pause by the screen reader.

input wrapped in label

Sounds like: Username, edit text.

No for and id attributes- don't do this!

Sounds like: edit text.

With for and id attributes

Sounds like: Username, edit text.

Required with aria-required="true"

Note: adding `required` marks the field as invalid in some screen readers.

Sounds like: Username, required, edit text.

Required with aria-required="true", error state using aria-invalid

Sounds like: Username, required, invalid data, edit text.

Error message in label

Sounds like: Username, Please enter your username, edit text, invalid data.

Error message after input, linked with aria-describedby

Please enter your username.

Sounds like: Username, edit text, invalid data. (pause) Please enter your username.

Error message before input, linked with aria-describedby

Please enter your username.

Sounds like: Username, edit text. (pause) Please enter your username.
